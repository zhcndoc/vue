# 社区指南 {#community-guide}

Vue 的社区正在以惊人的速度增长，如果你正在阅读这篇内容，那么你很可能已经准备好加入其中了。欢迎你！

现在，我们将回答社区能为你做什么，以及你能为社区做什么。

## 资源 {#resources}

### 行为准则 {#code-of-conduct}

我们的 [行为准则](/about/coc) 是一份指南，旨在让我们所有人以及我们参与其中的技术社区都更容易获得成长与丰富。

### 及时了解动态 {#stay-in-the-know}

- 关注我们的 [官方 Twitter 账号](https://x.com/vuejs)。
- 在 Twitter 或 GitHub 上关注我们的 [团队成员](./team)。
- 关注 [RFC 讨论](https://github.com/vuejs/rfcs)。
- 订阅 [官方博客](https://blog.vuejs.org/)。

### 获取支持 {#get-support}

- [Discord 聊天](https://discord.com/invite/vue)：Vue 开发者实时交流和聊天的地方。
- [论坛](https://forum.vuejs.org/)：向大家提问并获取有关 Vue 及其生态系统答案的最佳场所。
- [DEV Community](https://dev.to/t/vue)：在 Dev.to 上分享和讨论与 Vue 相关的话题。
- [见面会](https://events.vuejs.org/meetups)：想找到像你一样的本地 Vue 爱好者吗？有兴趣成为社区领袖吗？这里就有你需要的帮助和支持！
- [GitHub](https://github.com/vuejs)：如果你有 bug 需要报告或功能需要请求，这就是 GitHub issues 的用途。请遵守每个仓库 issue 模板中指定的规则。
- [Twitter 社区（非官方）](https://x.com/i/communities/1516368750634840064)：一个 Twitter 社区，你可以在这里结识其他 Vue 爱好者、寻求帮助，或者只是聊聊 Vue。

### 探索生态系统 {#explore-the-ecosystem}

- [Awesome Vue 页面](https://github.com/vuejs/awesome-vue)：看看其他优秀的人发布了哪些很棒的资源。
- [Vue Telescope Explorer](https://vuetelescope.com/explore)：探索使用 Vue 构建的网站，并了解它们使用了哪些框架/库。
- [Made with Vue.js](https://madewithvuejs.com/)：展示使用 Vue 构建的项目和库。
- [“Show and Tell” 子论坛](https://github.com/vuejs/core/discussions/categories/show-and-tell)：另一个很棒的地方，可以看看其他人用 Vue 生态系统为不断成长的 Vue 生态做了什么、构建了什么。

## 你可以做什么 {#what-you-can-do}

### 帮助其他用户 {#help-fellow-users}

贡献代码并不是为 Vue 社区做贡献的唯一形式。在 Discord 或论坛上回答其他 Vue 用户的问题，也被认为是一种有价值的贡献。

### 帮助整理 Issue {#help-triage-issues}

整理 issue 是指收集缺失的信息、运行复现、验证 issue 的有效性，以及调查 issue 的原因。

我们每天都会在 [GitHub 上的仓库](https://github.com/vuejs) 中收到大量 issue。与我们的用户数量相比，我们的带宽有限，因此仅 issue 整理一项就可能需要团队付出巨大的努力。通过帮助我们整理这些 issue，你就是在帮助我们变得更高效，从而让我们能够把时间花在更高优先级的工作上。

你不必以“修复它”为目标去整理 issue（不过那样当然也很好）。例如，分享你的调查结果，比如导致这个 bug 的 commit，就已经能为我们节省大量时间。

### 贡献代码 {#contribute-code}

贡献 bug 修复或新功能是你能做出的最直接的贡献形式。

Vue 核心仓库提供了一份 [贡献指南](https://github.com/vuejs/core/blob/main/.github/contributing.md)，其中包含了 pull request 指南，以及有关构建设置和高级架构的信息。其他子项目仓库也可能有自己的贡献指南——在提交 pull request 之前，请务必先阅读它们。

我们随时欢迎 bug 修复。对于新功能，最好先在 [RFC 仓库](https://github.com/vuejs/rfcs/discussions) 中讨论用例和实现细节。

### 分享（并构建）你的经验 {#share-and-build-your-experience}

除了在论坛和聊天中回答问题、分享资源之外，还有一些不那么明显的方式可以分享并扩展你所掌握的内容：

- **开发学习材料。** 常有人说，最好的学习方式就是教学。如果你正在用 Vue 做一些有趣的事情，可以通过撰写博客文章、制作工作坊，甚至发布一个你会在社交媒体上分享的 gist，来加深你的专业积累。
- **关注你关心的仓库。** 这会在该仓库有活动时向你发送通知，让你提前了解正在进行的讨论和即将到来的功能。这是建立专业知识的绝佳方式，最终你就能帮助处理 issue 和 pull request。

### 翻译文档 {#translate-docs}

我希望此刻，你正在用你偏好的语言阅读这句话。如果不是，你愿意帮助我们实现这一点吗？

有关你如何参与的更多细节，请参阅 [翻译指南](/translations/)。

### 成为社区领袖 {#become-a-community-leader}

你可以做很多事情来帮助 Vue 在你的社区中成长：

- **在本地见面会上发言。** 无论是做演讲还是主持工作坊，你都可以通过帮助新手和有经验的 Vue 开发者继续成长，为你的社区带来很多价值。
- **发起你自己的见面会。** 如果你所在地区还没有 Vue 见面会，你可以自己发起一个！使用 [events.vuejs.org 上的资源](https://events.vuejs.org/resources/#getting-started) 来帮助你成功！
- **帮助见面会组织者。** 在举办活动时，帮助永远不嫌多，所以请伸出援手帮助当地组织者，让每一场活动都取得成功。

如果你对如何更深入参与本地 Vue 社区有任何问题，请在 Twitter 上联系 [@vuejs_events](https://x.com/vuejs_events)！
